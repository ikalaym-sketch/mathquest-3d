# v0.28.0 強制紅隊測試

## 已驗證破壞情境

1. 同一天重複與同一居民交談，不可重複增加好感。
2. 同一天重複送禮，第二次必須拒絕且不扣物品。
3. 未解鎖或已完成的關係事件不可重複完成。
4. 商店休息日服務關閉，但居民仍可交談。
5. 雨天居民改走避雨位置，夜間生活 NPC 返回住家。
6. 新一週委託必須輪替，上一週接受狀態不得外溢。
7. 季節活動不在指定日期或物品不足時不可完成。
8. 初始夥伴只能選擇一次，非 Starter ID 必須拒絕。
9. 同一夥伴不可重複取得；錯誤 Boss／居民／區域事件不得誤解鎖。
10. 同一天同一夥伴互動不可重複取得好感。
11. 舊 Pet 必須映射至新夥伴並清除裝備欄 Pet，不得同時啟用雙 Runtime。
12. 8 個 GLB 均需通過 Header、9 個必要節點及 8 組有效動畫 Channel。
13. 非同行夥伴只能出現在農舍休息區，同行夥伴不得產生重複實例。
14. 兔子、狐狸、水獺、松鼠、小雞、小熊貓與狸貓的主要能力必須存在實際接線。

## 絕對會崩潰的三個後續風險

1. **直接替換最終夥伴 GLB，但破壞節點／動畫命名**：`CompanionActor` 將無法切換動畫或定位 Socket。最終美術必須遵守 Manifest 契約。
2. **擴充居民卻不新增取得／日程／對話驗證**：NPC 可能存在於資料但永遠不生成、沒有住家或送禮引用不存在物品。
3. **大量提高夥伴加成卻沒有實機經濟測試**：市場、星級產出與材料掉落會失去平衡，破壞農場與探索價值。
