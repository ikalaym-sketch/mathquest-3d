# v0.27.0 強制紅隊測試

## 已驗證破壞案例

1. 少一件防具時不得啟動四件套。
2. 混穿不同套裝不得啟動任一套裝能力。
3. 未持有裝備不得直接寫入裝備欄。
4. 武器不得裝入頭部／身體等錯誤欄位。
5. MP 不足時魔法攻擊不得執行或扣成負值。
6. 守護卷軸反射原始傷害；鏡盾不得反擊環境與 Boss 技能。
7. 無可成長作物時，成長粉不得消耗。
8. 星級農產品經背包、加工與出貨後不得遺失品質與單價。
9. 舊存檔重複作物 ID 遷移後數量不得減少；普通材料不得被誤移除。
10. 冬季不得在未解鎖溫室時種植非冬季作物。
11. 新農地未整地不得播種。
12. 使用錯誤工具不得執行農務；體力不足不得造成半寫入。
13. 同一動物同日不得重複撫摸或刷毛取得多次收益。
14. 沒有飼料時不得免費餵食。
15. 舊 `Date.now()` 加工資料遷移後必須改用世界分鐘，且移除舊完成時間。
16. 農莊升級缺少任一材料或金幣時不得開始施工。
17. 農莊升級開始後必須等待 1440 世界分鐘，不得即時升級。

## 仍可能崩潰的三個後續破綻

1. 若後續料理、委託或競賽直接讀舊 `materials`，會繞過品質商品實例；所有新功能必須只讀 `farmProducts`。
2. 若正式武器動畫未按武器類型建立 Socket／命中事件，資料效果雖正確，畫面仍可能出現傷害與動作不同步。
3. 若新增動物種類時不建立每日行為與產品 Handler，可能出現可互動但永遠無產出的空殼動物。
