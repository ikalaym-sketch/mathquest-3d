# MathQuest 3D v0.27.0｜Life Simulation Core Truth

## 基準

- 唯一基準：`v0.26.0`
- 基準 SHA-256：`2484d2bb2317207d04d1609f3a6469c1597d4a045d6ff870e1e7b814ce5e3770`
- 本版範圍：原改善藍圖的階段 A「裝備／道具 Runtime 真實化」與階段 B「正式農務操作與經濟」。

## 階段 A｜裝備與道具 Runtime 真實化

- 10 種消耗品全部改用正式 `Effect ID` 與集中 Handler；未支援效果不得消耗物品。
- 傳送卷軸改走單一場景轉場請求，避免直接改寫 Scene 狀態。
- 裝備前強制檢查持有權、裝備欄位與物品類型。
- 10 套防具必須四個正確部位同系列才啟動；Lv5 必須四件全部 Lv5。
- 10 套基礎／Lv5 效果接入實際傷害、移動、回復、農務、提示與掉落 Runtime。
- 20 把武器皆有正式 Lv5 Profile；MP 消耗、連擊、暴擊、吸血、反擊、穿透、散射、狀態與特殊傷害不再只存在描述。
- 修正狂戰士重複乘算、守護反射、鏡盾錯誤反擊環境／Boss 技能等語意漂移。
- 背包、商店、鐵匠、加工及套裝摘要主要操作完成繁體中文化。

## 階段 B｜正式農務與品質經濟

- 新農地必須先使用鋤頭整地，才可播種；舊存檔農田保持可用。
- 建立鋤頭、種子袋、澆水壺、鐮刀、刷具、飼料袋與撫摸等工具／行為契約。
- 所有農務動作消耗體力；工具等級可降低成本。
- 春、夏、秋、冬各 28 天；作物受適種季節限制，Lv5 星光溫室可跨季種植。
- 農產品改為具 `instanceId／quality／quantity／unitValue／harvestedWorldMinute／sourceType／metadata` 的正式實例。
- 一般、良好、優良、星級品質完整保留至背包、加工與出貨。
- 舊存檔中以種子 ID 表示的農產品會轉成品質實例，數量不遺失；其他材料不受影響。
- 動物加入飢餓、清潔、健康、心情、好感、每日餵食／撫摸／梳毛及品質產出。
- 營養飼料進入 Canonical DB、初始背包與商店；餵食必須實際消耗。
- 加工機由 `Date.now()` 改為 Canonical 世界分鐘，暫停及時間倍率不再造成時間來源分裂。
- 農莊升級改為金幣＋材料＋一個遊戲日施工，不再瞬間完成。
- 農夫四件套的增產與保水效果接入正式收成與土壤 Runtime。
