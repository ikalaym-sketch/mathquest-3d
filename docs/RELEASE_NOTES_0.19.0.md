# v0.19.0｜Region Gameplay Differentiation Engineering Build

## 版本定位

本版本直接延續已驗證的 `v0.18.0`，專注完成八區玩法差異化、區域事件狀態機、區域素材循環、結構 Socket 互動、四子區探索紀錄與每日秘境。依既定決策，`v0.17.0` 封版缺口仍延後處理，因此本包屬 **Engineering Build**。

實際檔案差異：`新增 51、修改 21、刪除 8（主要為舊 Dist Hash）、未變更 435`。

## 一、模組架構

### Canonical 資料

- `regionGameplayProfiles.js`：8 區、24 條事件、72 個事件步驟、16 個結構互動、8 種素材與 8 件紀念品。
- `regionMechanicChallenges.js`：24 個核心機制節點的一對一挑戰契約。
- `dailyRift.js`：依本地日期生成固定生態、遭遇與首勝獎勵。

### 獨立玩法模組

- 風之高地：角度旋轉風向標。
- 雪光谷：正負溫度調節。
- 農業平原：行列乘法陣列。
- 星光村外圍：數列規律。
- 水晶湖：鏡像與對稱。
- 日光峽谷：分段距離測量。
- 蘑菇濕地：倍數節拍敲擊。
- 機械遺跡：齒輪順序鎖。

八種玩法拆為八個獨立 React 元件，透過 `RegionMechanicChallengePanel` 路由；核心節點不再呼叫共用 Math Modal。

## 三、數據流 Dry Run

1. 玩家進入正式區域後，`RegionSubareaTracker` 依世界座標辨識子區。
2. 首次進入子區寫入探索紀錄；四區全探索只發放一次獎勵。
3. `RegionalEvent` 建立下一條未完成事件，事件只能依 Beacon → 機制／Socket → 蒐集／戰鬥順序推進。
4. 所有場景行為轉成 Signal，由 `RegionGameplayService` 驗證類型、目標與數量。
5. 一般／精英／Boss 固定掉落 1／2／4 個該區 Canonical 素材。
6. 4 個素材與 120G 可製作一次性區域紀念品。
7. 每日秘境由日期產生固定內容，只有首次擊敗守護者發放獎勵；一般怪不提供可重複刷取的金幣與掉落。

Rollback：回復 `v0.18.0` 工程包即可。新存檔欄位均具預設空物件，不需要 Migration。

## 四、UI 與操作推演

- 核心機制面板使用 `max-w-2xl`、`max-h-[92vh]` 與內容捲動，支援手機、平板、桌機。
- 正確完成前禁用完成寫入；成功後以單一按鈕返回場景。
- 齒輪順序按錯會重置；節拍錯誤不扣道具；所有錯誤均可直接重試。
- 已完成節點可再次同步後續事件，但不重複發放素材。
- 離開區域會清除暫時性的子區 HUD 狀態。

## 六、強制紅隊測試

1. **跨場景事件污染**：每日秘境、森林與試煉塔怪物若沿用目前區域 ID，會誤推進八區事件；本版已強制只有帶 `regionId` 的遭遇能送出區域戰鬥 Signal。
2. **預先完成節點卡關**：若玩家先完成機制、後接到引用該節點的事件，事件可能永遠無法前進；本版已允許完成節點重新送出 Signal，但不重複給獎。
3. **每日秘境重複刷取**：重新進場可重生一般怪；本版已關閉每日秘境一般怪金幣與素材掉落，正式獎勵只由日期首勝狀態控制。

## 七、明確未冒充完成

- 八區已具有獨立操作小遊戲，但尚未加入風場、冰面、橋索、折射光束等完整 3D 物理模擬。
- 24 條事件已是多步驟流程，但尚未加入完整劇情對話、過場與場景永久改造。
- 區域 Boss 仍未完成多階段技能與兒童友善戰鬥重建。
- Chromium 多裝置 WebGL 畫面仍受 `URLBlocklist=["*"]` 阻擋，不能標示實機通過。
- `v0.17.0` 封版缺口仍依指示延後。

## 八、三個必要測試情境

1. 每區依序完成三個機制，確認操作型態不同、素材只發放一次，完成節點仍能推進後接事件。
2. 每區完成三條事件，測試錯誤順序、離場重進、蒐集數量與一般怪擊殺數均不會跳步。
3. 同一天重進每日秘境，確認配置固定、首勝只領一次，完成後不再生成可刷取遭遇。
