# MathQuest 3D v0.10.0｜Phase 3 實作與審核紀錄

## 範圍

- 角色紙娃娃裝備 UI
- 裝備圖示與類型辨識
- 裝備比較與總能力
- 裝備外觀同步
- 小精靈 3D 跟隨與能力
- 手機 HUD 資訊收斂

## 實際操作結果

1. WebGL 2.0 正常建立，村莊與小精靈均實際渲染。
2. 木製短劍切換為獵人弓後：角色手持圖示、裝備槽、距離及總能力同步更新。
3. 小精靈探索感應實際回報 Balloon Altar 距離 9m。
4. 手機 390×844 實測：每日任務改為單一摘要，裝備頁先顯示角色及物品分類。
5. 九宮格地圖回歸測試仍為 9 個區域，裝備面板正常開啟。
6. 實際頁面測試未捕捉到 JavaScript page error。

## 能力規則

- 裝備能力以同一計算模組輸出，避免各 UI 使用不同公式。
- 防具防禦套用至怪物攻擊傷害，最低仍保留 1 點傷害。
- Healing 小精靈定時恢復生命。
- Swift 小精靈提高移動速度。
- Empower 小精靈依機率提高攻擊傷害。

## 已知限制／紅隊風險

1. 目前裝備圖示使用系統 Emoji，不同作業系統可能呈現不同；正式版需改為統一美術圖集。
2. 物品數量大幅增加後，裝備物品格需加入虛擬化或分頁，否則 DOM 數量會膨脹。
3. 紙娃娃目前是 UI 示意角色，尚未與未來捏娃娃的頭髮、膚色與表情資料模型整合。
4. 小精靈探索感應目前只找附近 POI，尚未計算路徑、障礙物及高低差。
5. 套裝效果仍有部分只存在資料描述，後續需逐一接入戰鬥／農場／學習流程。
